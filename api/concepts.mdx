---
title: List concepts
description: Retrieve all concepts accessible to the authenticated user including system and custom concepts
---

## List concepts

<CodeGroup>
```bash cURL
curl -X GET https://api.algomind.app/api/v1/concepts \
  -H "Authorization: Bearer YOUR_TOKEN"
```
</CodeGroup>

<ResponseExample>
```json Response
[
  {
    "id": 1,
    "user_id": null,
    "base_concept_id": null,
    "title": "Arrays",
    "description": "Fundamental data structure",
    "content": "Arrays are contiguous memory blocks that store elements of the same type...",
    "created_at": "2024-01-15T10:30:00Z"
  },
  {
    "id": 42,
    "user_id": "user_123",
    "base_concept_id": null,
    "title": "Dynamic Programming",
    "description": "My custom concept",
    "content": "Breaking down problems into overlapping subproblems...",
    "created_at": "2024-03-01T14:20:00Z"
  }
]
```
</ResponseExample>

### Endpoint

<ParamField path="method" type="string">
  GET
</ParamField>

<ParamField path="endpoint" type="string">
  /api/v1/concepts
</ParamField>

### Response

Returns an array of concept objects sorted by title in ascending order.

<ResponseField name="id" type="integer">
  The unique identifier of the concept.
</ResponseField>

<ResponseField name="user_id" type="string | null">
  The ID of the user who created this concept. `null` for system-wide concepts.
</ResponseField>

<ResponseField name="base_concept_id" type="integer | null">
  If this is a user override of a system concept, this contains the ID of the base system concept. Otherwise `null`.
</ResponseField>

<ResponseField name="title" type="string">
  The title of the concept.
</ResponseField>

<ResponseField name="description" type="string | null">
  Optional description of the concept.
</ResponseField>

<ResponseField name="content" type="string">
  The full content/explanation of the concept.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the concept was created.
</ResponseField>

<Note>
  The response includes:
  - All system concepts (where `user_id` is `null`) that the user hasn't overridden
  - All user-created concepts (where `user_id` matches the authenticated user)
  - User overrides of system concepts (where `base_concept_id` is not `null`)
</Note>

### Status codes

<ResponseField name="200" type="OK">
  Concepts successfully retrieved.
</ResponseField>

<ResponseField name="500" type="Internal Server Error">
  Failed to fetch concepts from the database.
</ResponseField>
## Create concept

Create a new custom concept for the authenticated user.

<CodeGroup>
```bash cURL
curl -X POST https://api.algomind.app/api/v1/concepts \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Backtracking",
    "description": "Exploring all possible solutions",
    "content": "Backtracking is an algorithmic technique..."
  }'
```
</CodeGroup>

### Request body

<ParamField body="title" type="string" required>
  The title of the concept.
</ParamField>

<ParamField body="description" type="string">
  Optional description of the concept.
</ParamField>

<ParamField body="content" type="string" required>
  The full content/explanation of the concept in markdown format.
</ParamField>

### Response

Returns the created concept object with a 201 status code.

<ResponseField name="id" type="integer">
  The unique identifier of the created concept.
</ResponseField>

<ResponseField name="user_id" type="string">
  The ID of the user who created this concept.
</ResponseField>

<ResponseField name="title" type="string">
  The title of the concept.
</ResponseField>

## Update concept

Update an existing concept. Only concepts you created can be updated.

<CodeGroup>
```bash cURL
curl -X PUT https://api.algomind.app/api/v1/concepts/42 \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Advanced Backtracking",
    "description": "Updated description",
    "content": "Updated content..."
  }'
```
</CodeGroup>

### Path parameters

<ParamField path="id" type="integer" required>
  The ID of the concept to update.
</ParamField>

### Request body

Same fields as create concept endpoint.

### Response

Returns the updated concept object with a 200 status code.

## Delete concept

Delete a concept you created. System concepts cannot be deleted.

<CodeGroup>
```bash cURL
curl -X DELETE https://api.algomind.app/api/v1/concepts/42 \
  -H "Authorization: Bearer YOUR_TOKEN"
```
</CodeGroup>

### Path parameters

<ParamField path="id" type="integer" required>
  The ID of the concept to delete.
</ParamField>

### Response

Returns 204 No Content on success.

<Note>
  Deleting a concept also deletes its review state if it exists.
</Note>

## Reset concept

Reset a user override back to the system concept.

<CodeGroup>
```bash cURL
curl -X POST https://api.algomind.app/api/v1/concepts/42/reset \
  -H "Authorization: Bearer YOUR_TOKEN"
```
</CodeGroup>

### Path parameters

<ParamField path="id" type="integer" required>
  The ID of the concept override to reset.
</ParamField>

### Response

Returns 200 OK with a success message when the override is deleted and you revert to the system concept.

<Warning>
  This only works on concept overrides (where base_concept_id is not null). It cannot reset system concepts.
</Warning>
