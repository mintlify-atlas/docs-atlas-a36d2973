## List concepts

<CodeGroup>
```bash cURL
curl -X GET https://api.algomind.app/api/v1/concepts \
  -H "Authorization: Bearer YOUR_TOKEN"
```
</CodeGroup>

### Endpoint

<ParamField path="method" type="string">
  GET
</ParamField>

<ParamField path="endpoint" type="string">
  /api/v1/concepts
</ParamField>

### Response

Returns an array of concept objects sorted by title in ascending order.

<ResponseField name="id" type="integer">
  The unique identifier of the concept.
</ResponseField>

<ResponseField name="user_id" type="string | null">
  The ID of the user who created this concept. `null` for system-wide concepts.
</ResponseField>

<ResponseField name="base_concept_id" type="integer | null">
  If this is a user override of a system concept, this contains the ID of the base system concept. Otherwise `null`.
</ResponseField>

<ResponseField name="title" type="string">
  The title of the concept.
</ResponseField>

<ResponseField name="description" type="string | null">
  Optional description of the concept.
</ResponseField>

<ResponseField name="content" type="string">
  The full content/explanation of the concept.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the concept was created.
</ResponseField>

<Note>
  The response includes:
  - All system concepts (where `user_id` is `null`) that the user hasn't overridden
  - All user-created concepts (where `user_id` matches the authenticated user)
  - User overrides of system concepts (where `base_concept_id` is not `null`)
</Note>

### Status codes

<ResponseField name="200" type="OK">
  Concepts successfully retrieved.
</ResponseField>

<ResponseField name="500" type="Internal Server Error">
  Failed to fetch concepts from the database.
</ResponseField>
