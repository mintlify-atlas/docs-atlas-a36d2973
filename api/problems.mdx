## Create problem

<CodeGroup>
```bash cURL
curl -X POST https://api.algomind.app/api/v1/problems \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "concept_id": 1,
    "title": "Two Sum",
    "link": "https://leetcode.com/problems/two-sum/",
    "difficulty": "EASY",
    "summary": "Find two numbers that add up to target",
    "description": "Given an array of integers nums and an integer target, return indices of the two numbers such that they add up to target.",
    "answer": "def twoSum(nums, target):\n    seen = {}\n    for i, num in enumerate(nums):\n        complement = target - num\n        if complement in seen:\n            return [seen[complement], i]\n        seen[num] = i",
    "answer_language": "python",
    "hints": "Use a hash map to store seen numbers"
  }'
```
</CodeGroup>

<ResponseExample>
```json Response
{
  "id": 42
}
```
</ResponseExample>

### Endpoint

<ParamField path="method" type="string">
  POST
</ParamField>

<ParamField path="endpoint" type="string">
  /api/v1/problems
</ParamField>

### Request body

<ParamField body="concept_id" type="integer" required>
  The ID of the concept this problem belongs to. Must be a concept accessible to the authenticated user.
</ParamField>

<ParamField body="title" type="string" required>
  The title of the problem.
</ParamField>

<ParamField body="link" type="string">
  Optional URL link to the original problem (e.g., LeetCode link).
</ParamField>

<ParamField body="difficulty" type="string" required>
  The difficulty level of the problem. Must be one of: `EASY`, `MEDIUM`, or `HARD`.
</ParamField>

<ParamField body="summary" type="string" required>
  A brief summary of the problem.
</ParamField>

<ParamField body="description" type="string">
  Full description of the problem including constraints and examples.
</ParamField>

<ParamField body="answer" type="string" required>
  Your solution code or explanation for solving the problem.
</ParamField>

<ParamField body="answer_language" type="string">
  The programming language used in the answer (e.g., "python", "javascript", "java").
</ParamField>

<ParamField body="hints" type="string">
  Hints to help solve the problem.
</ParamField>

### Response

<ResponseField name="id" type="integer">
  The unique identifier of the newly created problem.
</ResponseField>

<Note>
  When a problem is created, a review state is automatically initialized with:
  - `next_review_at` set to the current time
  - `interval_days` set to 0
  - `ease_factor` set to 2.5
  - `streak` set to 0
</Note>

### Status codes

<ResponseField name="201" type="Created">
  Problem successfully created.
</ResponseField>

<ResponseField name="400" type="Bad Request">
  Invalid request body or concept_id does not exist.
</ResponseField>

<ResponseField name="403" type="Forbidden">
  The specified concept is not accessible to the authenticated user.
</ResponseField>

<ResponseField name="500" type="Internal Server Error">
  Failed to create problem or review state.
</ResponseField>
## Get all problems

Retrieve all problems created by the authenticated user.

<CodeGroup>
```bash cURL
curl -X GET https://api.algomind.app/api/v1/problems \
  -H "Authorization: Bearer YOUR_TOKEN"
```
</CodeGroup>

### Response

Returns an array of problem objects.

<ResponseField name="id" type="integer">
  The unique identifier of the problem.
</ResponseField>

<ResponseField name="user_id" type="string">
  The ID of the user who created the problem.
</ResponseField>

<ResponseField name="concept_id" type="integer">
  The ID of the associated concept.
</ResponseField>

<ResponseField name="title" type="string">
  The title of the problem.
</ResponseField>

<ResponseField name="difficulty" type="string">
  Problem difficulty: EASY, MEDIUM, or HARD.
</ResponseField>

<ResponseField name="summary" type="string">
  Brief summary of the problem.
</ResponseField>

<ResponseField name="answer" type="string">
  Your solution to the problem.
</ResponseField>

## Get problem by ID

Retrieve a specific problem by its ID.

<CodeGroup>
```bash cURL
curl -X GET https://api.algomind.app/api/v1/problems/123 \
  -H "Authorization: Bearer YOUR_TOKEN"
```
</CodeGroup>

### Path parameters

<ParamField path="problem_id" type="integer" required>
  The ID of the problem to retrieve.
</ParamField>

### Response

Returns a single problem object with the same fields as the create response.

## Delete problem

Delete a problem you created.

<CodeGroup>
```bash cURL
curl -X DELETE https://api.algomind.app/api/v1/problems/123 \
  -H "Authorization: Bearer YOUR_TOKEN"
```
</CodeGroup>

### Path parameters

<ParamField path="problem_id" type="integer" required>
  The ID of the problem to delete.
</ParamField>

### Response

Returns 204 No Content on success.

<Warning>
  Deleting a problem also deletes its review state and review logs.
</Warning>

## Add problem to review queue

Manually add a problem back to your review queue.

<CodeGroup>
```bash cURL
curl -X POST https://api.algomind.app/api/v1/problems/add-to-review-queue/123 \
  -H "Authorization: Bearer YOUR_TOKEN"
```
</CodeGroup>

### Path parameters

<ParamField path="problem_id" type="integer" required>
  The ID of the problem to add to the review queue.
</ParamField>

### Response

Returns 200 OK with a success message. The problem's next review date is set to now, making it immediately reviewable.
