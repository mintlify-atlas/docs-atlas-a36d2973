This guide will walk you through setting up Algomind on your local machine for development.

## Prerequisites

Before you begin, make sure you have the following installed:

- **Node.js 20** or later
- **Go 1.24** or later
- **PostgreSQL 15** or later
- **golang-migrate CLI** for running database migrations

<Warning>
  You'll also need a Clerk account to set up authentication. Sign up at [clerk.com](https://clerk.com) if you don't have one.
</Warning>

## Clone the repository

Start by cloning the Algomind repository:

```bash
git clone https://github.com/VatsalP117/algomind.git
cd algomind
```

## Set up the database

<Steps>
  <Step title="Create a PostgreSQL database">
    Create a new PostgreSQL database for Algomind:

    ```bash
    createdb algomind
    ```

    Or using SQL:

    ```sql
    CREATE DATABASE algomind;
    ```
  </Step>

  <Step title="Run database migrations">
    Navigate to the backend directory and run migrations:

    ```bash
    cd algomind-backend
    migrate -path ./migrations -database "$DATABASE_URL" up
    ```

    Replace `$DATABASE_URL` with your actual connection string:

    ```
    postgresql://username:password@localhost:5432/algomind?sslmode=disable
    ```
  </Step>
</Steps>

<Info>
  The migrations will create five tables: `users`, `concepts`, `problems`, `review_states`, and `review_logs`. Check out the [architecture](/architecture#database-schema) page for schema details.
</Info>

## Configure environment variables

### Backend configuration

Create a `.env` file in the `algomind-backend` directory:

```bash
cd algomind-backend
cp .env.example .env
```

Fill in the following variables:

```bash
DATABASE_URL=postgresql://username:password@localhost:5432/algomind?sslmode=disable
CLERK_SECRET_KEY=sk_test_...
PORT=8080
```

<Accordion title="Where to find your Clerk secret key">
  1. Go to your [Clerk dashboard](https://dashboard.clerk.com)
  2. Select your application
  3. Navigate to **API Keys** in the sidebar
  4. Copy the **Secret Key** (starts with `sk_test_` for development)
</Accordion>

### Frontend configuration

Create a `.env.local` file in the `algomind-frontend` directory:

```bash
cd ../algomind-frontend
cp .env.example .env.local
```

Fill in the following variables:

```bash
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_...
NEXT_PUBLIC_SITE_URL=http://localhost:3000
NEXT_PUBLIC_CLERK_SIGN_IN_URL=/sign-in
NEXT_PUBLIC_CLERK_SIGN_UP_URL=/sign-up
NEXT_PUBLIC_CLERK_AFTER_SIGN_IN_URL=/dashboard
NEXT_PUBLIC_CLERK_AFTER_SIGN_UP_URL=/dashboard
NEXT_PUBLIC_API_URL=http://localhost:8080
```

<Note>
  All `NEXT_PUBLIC_*` variables are baked into the frontend bundle at build time, so you'll need to rebuild if you change them.
</Note>

## Start the backend

From the `algomind-backend` directory:

```bash
go run ./cmd/api/main.go
```

You should see output like:

```
Connected to database
Server started on :8080
```

The backend API will be available at `http://localhost:8080`.

<Tip>
  Test the health endpoint to verify the server is running:
  
  ```bash
  curl http://localhost:8080/health
  ```
</Tip>

## Start the frontend

Open a new terminal and navigate to the `algomind-frontend` directory:

<Steps>
  <Step title="Install dependencies">
    ```bash
    npm install
    ```
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run dev
    ```

    The frontend will be available at `http://localhost:3000`.
  </Step>
</Steps>

## Common issues

<Accordion title="Database connection failed">
  Make sure PostgreSQL is running and your `DATABASE_URL` is correct:

  ```bash
  psql -d algomind -c "SELECT 1;"
  ```

  If this fails, check your PostgreSQL service status.
</Accordion>

<Accordion title="Clerk authentication error">
  Verify that:
  - Your Clerk API keys are correct
  - The publishable key in `.env.local` matches your Clerk dashboard
  - The secret key in `.env` matches your Clerk dashboard
  - You're using the correct environment (development vs. production)
</Accordion>

<Accordion title="Frontend can't reach backend">
  Ensure:
  - The backend is running on port 8080
  - `NEXT_PUBLIC_API_URL` in `.env.local` points to `http://localhost:8080`
  - There are no CORS issues (the backend should allow requests from `localhost:3000`)
</Accordion>
