Algomind implements a modified SM-2 (SuperMemo 2) algorithm to intelligently schedule your problem reviews. The algorithm adjusts review intervals based on how well you recall each problem, ensuring you spend time on what matters most.

## How it works

Every time you review a problem, you rate your recall quality. The algorithm then calculates:

- **Next review date**: When you'll see this problem again
- **Interval days**: How many days until the next review
- **Ease factor**: A multiplier that affects future intervals (tracks how "easy" this problem is for you)
- **Streak**: Consecutive successful reviews

## Rating system

When you complete a problem review, you choose one of four ratings:

<Steps>
  <Step title="Again (Failed recall)">
    You couldn't solve the problem or needed significant help.
    
    **Algorithm response:**
    - Review interval resets to 0 days (you'll see it again today)
    - Streak resets to 0
    - Ease factor decreases by 0.20 (minimum 1.3)
  </Step>
  
  <Step title="Hard (Difficult recall)">
    You solved it, but it was challenging and you struggled.
    
    **Algorithm response:**
    - Interval increases by 20% (minimum 1 day for new problems)
    - Streak increments by 1
    - Ease factor decreases by 0.15 (minimum 1.3)
  </Step>
  
  <Step title="Good (Normal recall)">
    You solved it with reasonable effort—the expected difficulty level.
    
    **Algorithm response:**
    - Interval multiplies by current ease factor
    - Streak increments by 1
    - Ease factor stays the same
  </Step>
  
  <Step title="Easy (Perfect recall)">
    You solved it quickly and confidently with no issues.
    
    **Algorithm response:**
    - Interval multiplies by ease factor × 1.3
    - Minimum 4 days for new problems
    - Streak increments by 1
    - Ease factor increases by 0.15
  </Step>
</Steps>

## Implementation details

The core algorithm is implemented in the backend's SRS (Spaced Repetition System) package:

```go algomind-backend/internal/srs/algo.go
func CalculateReview(rating string, currentInterval int, currentEase float64, currentStreak int) ReviewResult {
    var newInterval int
    var newEase = currentEase
    var newStreak = currentStreak

    switch rating {
    case "AGAIN":
        newInterval = 0
        newStreak = 0
        newEase = math.Max(1.3, currentEase-0.20)

    case "HARD":
        newInterval = int(float64(currentInterval) * 1.2)
        if newInterval == 0 {
            newInterval = 1
        }
        newEase = math.Max(1.3, currentEase-0.15)
        newStreak++

    case "GOOD":
        newInterval = int(float64(currentInterval) * currentEase)
        if newInterval == 0 {
            newInterval = 1
        }
        newStreak++

    case "EASY":
        newInterval = int(float64(currentInterval) * currentEase * 1.3)
        if newInterval == 0 {
            newInterval = 4
        }
        newEase += 0.15
        newStreak++
    }

    nextReview := time.Now().AddDate(0, 0, newInterval)

    return ReviewResult{
        NextReviewAt: nextReview,
        IntervalDays: newInterval,
        EaseFactor:   newEase,
        Streak:       newStreak,
    }
}
```

<Note>
  The ease factor has a floor of 1.3 to prevent intervals from becoming too short, even for very difficult problems.
</Note>

## Interval progression examples

### Consistent "Good" ratings

Assuming a starting ease factor of 2.5:

- Review 1: 0 days → 1 day (ease: 2.5)
- Review 2: 1 day → 2 days (ease: 2.5)
- Review 3: 2 days → 5 days (ease: 2.5)
- Review 4: 5 days → 12 days (ease: 2.5)
- Review 5: 12 days → 30 days (ease: 2.5)

### Mix of "Easy" and "Hard" ratings

- Review 1: "EASY" → 4 days (ease: 2.65)
- Review 2: "EASY" → 13 days (ease: 2.80)
- Review 3: "HARD" → 15 days (ease: 2.65)
- Review 4: "GOOD" → 39 days (ease: 2.65)

### Failed review impact

- Review streak: 5 problems
- Next: "AGAIN" rating
- Result: Back to 0 days, streak resets, ease factor drops

<Tip>
  Be honest with your ratings! The algorithm works best when you accurately reflect your recall quality. Rating "Good" when you actually struggled will lead to premature intervals.
</Tip>

## Streak tracking

Your **streak** represents consecutive successful reviews (any rating except "AGAIN"). Streaks:

- Increment on "HARD", "GOOD", or "EASY" ratings
- Reset to 0 on "AGAIN" ratings
- Are tracked per problem (not globally)
- Help you identify which problems you've mastered

<Info>
  High streaks combined with long intervals indicate strong retention. These problems are candidates for archiving or reducing review frequency.
</Info>

## Why SM-2?

The SM-2 algorithm has been used successfully for decades in spaced repetition software like Anki and SuperMemo. Its key advantages:

- **Adaptive**: Adjusts to your individual learning patterns
- **Efficient**: Reduces review burden as mastery increases
- **Proven**: Backed by cognitive science research on memory retention
- **Simple**: Easy to understand and debug

For coding interview prep, this means you'll naturally spend more time on weak areas while maintaining long-term retention of problems you've mastered.
